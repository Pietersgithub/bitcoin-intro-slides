## What is the Blockchain? - Distributed Database. - Records are "Blocks" of transactions, chained together with cryptographic hash. - Blocks are: - Tamper-proof (verifiable) - Immutable (permanent) Blockchain is a distributed Ledger containing the full history of all bitcoins since the genesis block (3rd Jan 2009).
## What's new? The first time in history we have a digital asset that can be transferred but cannot be duplicated. - Solves the problem of "double spending". - Decentralized - Trustless: no financial institutions, governments... - Censorship resistant. - Resilient and Secure. - Permissionless: instant low cost bank for everyone 24/7 365 days/year. - Transnational, borderless.

## Criptographic Hash functions Hash functions maps data of arbitrary size to a fixed size. ![Hash function](images/hash-functions.png) They are used to create "digests" or "fingerprints" for data integrity. Note: - Un esempio di controllo di integrità è il controllo di parità o il check-sum. - Un altro esempio è il codice fiscale, l'ultima lettera è calcolata sulla base di tutte le altre.
## Cryptographic Hash functions Properties: - Deterministic. - Quick to compute. - Infeasible to reverse. - Small change on the input => complete different hash value. - Resistant to collision. Popular hash functions: MD5, SHA, DSA. Bitcoin uses SHA-256 for the "Proof of Work". Note: - deterministic: dato un input viene prodotto sempre lo stesso output - infeasible to reverse: dev'essere impossibile, dato un hash ottenere il messaggio originale L'unico modo dev'essere il brute force. - small change: idealmente, cambiando un bit nel messaggio originale ogni bit nel valore dell'hash dovrebbe avere il 50% di possibilità di cambiare. - collision: due input diversi non devono mai generare lo stesso hash - SHA: Secure Hash Algorithm - DSA: Digital Signature Algorithm - MD5: Message Digest
